### 二、前端具体要求

**技术栈**：Vue 3.4 + TypeScript 5 + Vite 5 + Pinia 2 + Vue Router 4 + Axios + Tailwind CSS 3 + Element Plus

**配色方案（Tailwind 扩展）**：
```js
colors: {
  primary: '#000000',
  background: '#FFFFFF',
  surface: '#F5F5F5',
  accent: '#2563EB'
}
```

**预置文件清单**：

| 文件 | 必须包含的内容 |
|------|---------------|
| `src/utils/request.ts` | Axios 实例：baseURL 读环境变量、请求拦截器带 Token、响应拦截器处理 401 |
| `src/api/user.ts` | 登录 `/api/v1/auth/login`、注册 `/api/v1/auth/register`、获取用户信息 |
| `src/stores/auth.ts` | Pinia Store：token、userInfo、login、logout、isAuthenticated |
| `src/router/index.ts` | 常量路由表、路由守卫（未登录跳/login） |
| `src/views/Login.vue` | 黑白极简风格登录页：用户名+密码+登录按钮，登录成功跳首页 |
| `src/views/Home.vue` | 首页占位，展示登录后的用户信息 |
| `src/views/404.vue` | 简洁 404 页面 |
| `vite.config.ts` | `@` 别名指向 `src`、开发代理 `/api` 到 `http://localhost:8080` |

**ESLint + Prettier 规则**：
- Vue3 推荐规则 + TypeScript 严格规则
- 禁止 `any` 类型
- 单引号、2 空格缩进、无尾随分号
- 函数超过 50 行警告

### 三、后端具体要求

**技术栈**：Go 1.22 + Gin 1.9 + GORM 1.25 + PostgreSQL（pgx 驱动）+ Redis（go-redis）+ Viper + Zap + JWT（golang-jwt/jwt/v5）

**预置文件清单**：

| 文件 | 必须包含的内容 |
|------|---------------|
| `cmd/main.go` | 初始化：配置加载 → Zap日志 → GORM连接 → Redis连接 → Gin引擎 → 路由注册 → 启动 |
| `config/config.go` | 结构体映射：Server.Port、Database.*、Redis.*、JWT.Secret/Expire |
| `internal/handler/user_handler.go` | 注册、登录、获取当前用户、刷新Token四个Handler |
| `internal/service/user_service.go` | 对应四个Service接口（空壳实现，直接调Repository） |
| `internal/repository/user_repository.go` | 增删改查基础方法（空壳） |
| `internal/model/user.go` | User 结构体：ID、Username、PasswordHash、Email、AvatarURL、Status、CreatedAt、UpdatedAt、DeletedAt |
| `internal/middleware/jwt.go` | JWT 验证中间件，解析 `Authorization: Bearer <token>` |
| `internal/middleware/cors.go` | 允许前端域名跨域 |
| `internal/middleware/logger.go` | Zap 请求日志（含 TraceID） |
| `pkg/response/response.go` | `Success(c, data)`、`Error(c, code, msg)` 两个函数，统一格式 `{code,data,msg,request_id}` |
| `pkg/jwt/jwt.go` | `GenerateToken(userID)`、`ParseToken(tokenString)`、`GenerateRefreshToken()` |
| `scripts/init_db.sql` | `CREATE DATABASE template_db; CREATE TABLE users (...)` |

**环境变量模板（`.env.example`）**：
```env
# Server
SERVER_PORT=8080
SERVER_MODE=debug

# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=template_db
DB_SSLMODE=disable

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# JWT
JWT_SECRET=your-256-bit-secret-key-here
JWT_ACCESS_EXPIRE=2h
JWT_REFRESH_EXPIRE=168h
```

**GORM 模型规范**：
- 必须内嵌 `gorm.Model`（ID、CreatedAt、UpdatedAt、DeletedAt）
- 字段加 `comment:"说明"` 标签
- 密码字段命名 `password_hash`，存 bcrypt 结果
- 状态字段 `status` tinyint，默认 1

### 五、交付标准（跑起来 checklist）

生成完成后，必须验证：

- [ ] `cd frontend && npm install && npm run dev` → 浏览器打开 `http://localhost:5173` 看到登录页
- [ ] `cd backend && cp .env.example .env && go mod tidy && go run cmd/main.go` → 后端监听 `:8080`
- [ ] 前端点击登录，请求能发到后端 `/api/v1/auth/login`
- [ ] 后端 Swagger UI 可访问：`http://localhost:8080/swagger/index.html`
- [ ] 前端 `.env.production` 中 `VITE_API_BASE_URL` 指向 `/api`
- [ ] 后端 CORS 允许前端开发域名
- [ ] `go build` 能编译出单二进制文件
- [ ] `npm run build` 能生成 `dist/` 目录
