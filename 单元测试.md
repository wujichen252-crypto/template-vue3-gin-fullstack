## 一、核心思路

**你不是让 AI "随便写点测试"，而是让 AI 扮演"懂你的技术栈的测试工程师"。**

关键要告诉 AI 四件事：

1. **你是谁**（Go/Gin、Django-Ninja 还是 Spring Boot）
2. **测什么**（函数签名、输入输出、边界条件）
3. **用什么测**（ testify、pytest-django、JUnit+Mockito）
4. **输出什么**（文件路径、命名规范、覆盖率目标）

  - 必须包含正常路径 + 异常路径 + 边界条件
  - 不允许测试代码依赖真实数据库/外部网络

【待测代码】
```

{把你的源码粘贴到这里}

```
【输出要求】
1. 文件路径：{按模板仓库规范，如 backend/internal/service/note_test.go}
2. 命名规范：{Go 用 TestXxx，Python 用 test_xxx，Java 用 xxxTest}

## 四、向 AI 提供代码的标准格式

**不要只丢一个函数给 AI，至少提供以下上下文：**

```

【模块职责】
这个文件是做什么的（如：用户笔记的 CRUD 接口、AI 摘要生成服务）

【依赖接口/结构体】

```go
// 如果是 Go，给出 interface 定义
type NoteStore interface {
    Create(ctx context.Context, note *Note) error
    GetByID(ctx context.Context, id string) (*Note, error)
}
```

【被测函数】

```go
func (s *Service) GenerateSummary(ctx context.Context, content string) (string, error) {
    if len(content) == 0 {
        return "", ErrEmptyContent
    }
    // 调用 AI 接口...
    return summary, nil
}
```

【边界条件说明】

- content 最大长度 5000 字符
- AI 接口可能返回 429 限流错误
- 空字符串应返回 ErrEmptyContent

```

## 五、AI 输出规范要求

复制以下内容给 AI，约束其输出格式：

```

【输出格式规范】

1. 只输出代码，不要解释每行在干什么（除非逻辑复杂需要注释）
2. 代码块标注语言类型（```go / ```python / ```java）
3. 文件路径用注释写在第一行：// file: backend/internal/service/note_test.go
4. 如果需要用外部依赖，在代码顶部列出 import / require 清单
5. 确保代码可以直接复制到项目中运行（不要缺 import、不要语法错误）
6. 如果待测代码有 bug 导致无法测试，先指出问题再给出测试

```

## 六、三种典型场景的完整示例

### 6.1 示例 A：Go / Gin —— AI 摘要服务

**你给 AI 的输入：**

```

【角色】Go 测试工程师
【技术约束】使用 testify，文件 backend/internal/service/ai_test.go
【待测代码】

```go
package service

import (
    "context"
    "errors"
)

type AISummaryClient interface {
    Summarize(ctx context.Context, text string) (string, error)
}

type AIService struct {
    client AISummaryClient
    maxLen int
}

func NewAIService(client AISummaryClient) *AIService {
    return &AIService{client: client, maxLen: 5000}
}

func (s *AIService) GenerateSummary(ctx context.Context, content string) (string, error) {
    if content == "" {
        return "", errors.New("content is empty")
    }
    if len(content) > s.maxLen {
        content = content[:s.maxLen]
    }
    return s.client.Summarize(ctx, content)
}
```

【输出要求】正常路径、空字符串错误、超长截断、AI 客户端错误

```
**AI 应返回：**

```go
// file: backend/internal/service/ai_test.go
package service

import (
	"context"
	"errors"
	"testing"

	"github.com/stretchr/testify/assert"
	"github.com/stretchr/testify/mock"
)

// Mock 实现
type mockAIClient struct {
	mock.Mock
}

func (m *mockAIClient) Summarize(ctx context.Context, text string) (string, error) {
	args := m.Called(ctx, text)
	return args.String(0), args.Error(1)
}

func TestAIService_GenerateSummary(t *testing.T) {
	client := new(mockAIClient)
	svc := NewAIService(client)

	t.Run("happy path", func(t *testing.T) {
		client.On("Summarize", mock.Anything, "hello world").
			Return("summary", nil).Once()

		res, err := svc.GenerateSummary(context.Background(), "hello world")
		assert.NoError(t, err)
		assert.Equal(t, "summary", res)
	})

	t.Run("empty content", func(t *testing.T) {
		res, err := svc.GenerateSummary(context.Background(), "")
		assert.Error(t, err)
		assert.Equal(t, "", res)
		client.AssertNotCalled(t, "Summarize")
	})

	t.Run("truncates long content", func(t *testing.T) {
		longContent := make([]byte, 6000)
		for i := range longContent {
			longContent[i] = 'a'
		}
		expected := string(longContent[:5000])

		client.On("Summarize", mock.Anything, expected).
			Return("long summary", nil).Once()

		res, err := svc.GenerateSummary(context.Background(), string(longContent))
		assert.NoError(t, err)
		assert.Equal(t, "long summary", res)
	})

	t.Run("client error", func(t *testing.T) {
		client.On("Summarize", mock.Anything, "fail").
			Return("", errors.New("ai overload")).Once()

		res, err := svc.GenerateSummary(context.Background(), "fail")
		assert.Error(t, err)
		assert.Contains(t, err.Error(), "ai overload")
		assert.Equal(t, "", res)
	})
}
```

### 6.2 示例 B：Django-Ninja —— 笔记 CRUD

**你给 AI 的输入：**

```
【角色】Django-Ninja 测试工程师
【技术约束】pytest + ninja.testing.TestClient，文件 backend/neuramind/api/tests/test_notes.py
【待测代码】
```python
# backend/neuramind/api/notes.py
from ninja import Router
from django.shortcuts import get_object_or_404
from .models import Note
from .schemas import NoteIn, NoteOut

router = Router()

@router.post("/notes", response=NoteOut)
def create_note(request, payload: NoteIn):
    note = Note.objects.create(user=request.auth, **payload.dict())
    return note

@router.get("/notes/{note_id}", response=NoteOut)
def get_note(request, note_id: int):
    note = get_object_or_404(Note, id=note_id, user=request.auth)
    return note
```

【模型】Note(user, title, content, created_at)
【认证】request.auth 返回当前 User 实例
【输出要求】创建成功、获取成功、获取他人笔记 404、未认证 401

```
**AI 应返回：**

```python
# file: backend/neuramind/api/tests/test_notes.py
import pytest
from ninja.testing import TestClient
from django.contrib.auth import get_user_model

from neuramind.api.notes import router

User = get_user_model()
client = TestClient(router)

@pytest.fixture
def user(db):
    return User.objects.create_user(username="tester", password="123")

@pytest.fixture
def other_user(db):
    return User.objects.create_user(username="other", password="123")

@pytest.fixture
def note(db, user):
    from neuramind.api.models import Note
    return Note.objects.create(user=user, title="My Note", content="Content")

@pytest.mark.django_db
class TestCreateNote:
    def test_create_success(self, user):
        # 模拟认证用户
        response = client.post("/notes", json={"title": "New", "content": "Body"}, auth=user)
        assert response.status_code == 200
        data = response.json()
        assert data["title"] == "New"
        assert data["id"] is not None

    def test_unauthenticated(self):
        response = client.post("/notes", json={"title": "New", "content": "Body"})
        assert response.status_code in (401, 403)

@pytest.mark.django_db
class TestGetNote:
    def test_get_own_note(self, user, note):
        response = client.get(f"/notes/{note.id}", auth=user)
        assert response.status_code == 200
        assert response.json()["title"] == "My Note"

    def test_get_others_note_returns_404(self, other_user, note):
        response = client.get(f"/notes/{note.id}", auth=other_user)
        assert response.status_code == 404
```

### 6.3 示例 C：Spring Boot —— 用户服务

**你给 AI 的输入：**

```
【角色】Spring Boot 测试工程师
【技术约束】JUnit 5 + Mockito，纯 Service 层测试，文件 backend/src/test/java/com/neuramind/service/UserServiceTest.java
【待测代码】
```java
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private PasswordEncoder passwordEncoder;

    public User register(String username, String rawPassword) {
        if (userRepository.existsByUsername(username)) {
            throw new BusinessException("用户名已存在");
        }
        User user = new User();
        user.setUsername(username);
        user.setPassword(passwordEncoder.encode(rawPassword));
        return userRepository.save(user);
    }
}
```

【输出要求】注册成功、用户名重复抛异常、密码被加密存储，使用 BDD 注释

```
**AI 应返回：**

```java
// file: backend/src/test/java/com/neuramind/service/UserServiceTest.java
package com.neuramind.service;

import com.neuramind.entity.User;
import com.neuramind.exception.BusinessException;
import com.neuramind.repository.UserRepository;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;
import org.springframework.security.crypto.password.PasswordEncoder;

import static org.assertj.core.api.Assertions.assertThat;
import static org.assertj.core.api.Assertions.assertThatThrownBy;
import static org.mockito.ArgumentMatchers.any;
import static org.mockito.BDDMockito.given;
import static org.mockito.Mockito.never;
import static org.mockito.Mockito.verify;

@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @Mock
    private PasswordEncoder passwordEncoder;

    @InjectMocks
    private UserService userService;

    @Test
    void shouldRegisterNewUserSuccessfully() {
        // given
        given(userRepository.existsByUsername("alice")).willReturn(false);
        given(passwordEncoder.encode("123456")).willReturn("encoded_pass");
        given(userRepository.save(any(User.class))).willAnswer(inv -> {
            User u = inv.getArgument(0);
            u.setId(1L);
            return u;
        });

        // when
        User result = userService.register("alice", "123456");

        // then
        assertThat(result.getId()).isEqualTo(1L);
        assertThat(result.getUsername()).isEqualTo("alice");
        assertThat(result.getPassword()).isEqualTo("encoded_pass");
        verify(userRepository).save(any(User.class));
    }

    @Test
    void shouldThrowWhenUsernameExists() {
        // given
        given(userRepository.existsByUsername("bob")).willReturn(true);

        // when & then
        assertThatThrownBy(() -> userService.register("bob", "123456"))
                .isInstanceOf(BusinessException.class)
                .hasMessageContaining("用户名已存在");

        verify(userRepository, never()).save(any());
    }
}
```

## 七、质量检查清单

拿到 AI 生成的测试后，用以下清单快速验收：

| 检查项                            | Go                | Python             | Java                | 是否通过 |
| --------------------------------- | ----------------- | ------------------ | ------------------- | -------- |
| 文件路径符合模板仓库规范          | `*_test.go` 同包  | `tests/test_*.py`  | `src/test/java/...` | ☐        |
| 能直接运行不报错                  | `go test ./...`   | `pytest`           | `mvn test`          | ☐        |
| 包含正常路径                      | ☐                 | ☐                  | ☐                   | ☐        |
| 包含异常/错误路径                 | ☐                 | ☐                  | ☐                   | ☐        |
| 包含边界条件                      | ☐                 | ☐                  | ☐                   | ☐        |
| 外部依赖已 Mock/隔离              | mock interface    | mock ORM/HTTP      | Mockito mock        | ☐        |
| 没有调用真实数据库/网络           | ☐                 | ☐                  | ☐                   | ☐        |
| 断言具体（不是 assertTrue(true)） | assert.Equal      | assert ==          | assertThat          | ☐        |
| 测试名能说明测什么场景            | `TestXxx/xxxCase` | `test_xxx_success` | `shouldXxxWhenYyy`  | ☐        |

**如果 AI 生成的测试缺了某一项，把对应行复制给 AI：**

> "补充异常路径测试：当 AI 接口返回 429 限流时，应该返回什么错误"
> "把 mock 改成使用 unittest.mock 而不是外部库，减少依赖"
> "使用表驱动测试，把 3 个相似 case 合并"

## 八、进阶技巧

### 8.1 让 AI 批量生成

如果你有 5 个 Service 文件，可以这样组织：

```
【批量任务】
请为以下 5 个模块各生成测试文件，统一放在 backend/internal/service/ 下：

1. user_service.go —— 用户注册/登录
2. note_service.go —— 笔记 CRUD
3. ai_service.go —— AI 摘要生成
4. tag_service.go —— 标签管理
5. share_service.go —— 分享链接生成

【统一约束】
- 全部使用 testify/mock
- 每个文件至少 3 个 test case
- 统一输出格式：```go 代码块 + 文件路径注释
```

## 总结

| 步骤 | 你给 AI 什么        | AI 给你什么              |
| ---- | ------------------- | ------------------------ |
| 1    | 角色定义 + 技术约束 | 符合规范的测试框架代码   |
| 2    | 待测源码 + 依赖接口 | 带 Mock 的隔离测试       |
| 3    | 边界条件说明        | 正常/异常/边界三类 Case  |
| 4    | 输出格式规范        | 可直接复制粘贴运行的文件 |

> **核心心法**：把 AI 当成"刚入职的测试工程师" —— 你必须告诉它项目规范、技术栈、测什么、怎么测，它才能产出可用的代码。给的上下文越完整，生成的测试越靠谱。
