# pnpm 最佳实践手册

> **适用场景**：Vue3 + Vite 前端工程、Node.js 后端、Monorepo 全栈项目  
> **版本参考**：pnpm v9.x  
> **归档日期**：2026-04-24

---

## 一、为什么选 pnpm

| 痛点                           | npm/yarn                         | pnpm 解决方案                                             |
| ------------------------------ | -------------------------------- | --------------------------------------------------------- |
| `node_modules` 体积爆炸        | 每个项目独立复制依赖包           | **全局内容寻址存储**，项目内全是硬链接，磁盘占用减少 70%+ |
| 幽灵依赖（Ghost Dependencies） | 扁平化结构导致未声明的包可被引用 | **严格依赖隔离**，非直接依赖无法 `import`                 |
| 安装速度慢                     | 串行/半并行下载                  | 二次安装秒级完成，CI/CD 构建显著提速                      |
| Monorepo 支持弱                | Workspaces 功能简陋              | 原生 workspace + 强大的过滤命令 + 依赖提升控制            |

## 二、安装与初始化

### 2.1 安装 pnpm

```bash
# 方式1：通过 npm（最快捷）
npm install -g pnpm

# 方式2：通过官方脚本（推荐，避免 npm 缓存污染）
curl -fsSL https://get.pnpm.io/install.sh | sh -

# 方式3：通过 Homebrew（macOS）
brew install pnpm

# 验证
pnpm -v
```

### 2.2 配置镜像（国内网络）

```bash
# 设置淘宝镜像（或公司私有仓库）
pnpm config set registry https://registry.npmmirror.com

# 查看当前配置
pnpm config list
```

### 2.3 项目初始化

```bash
# 进入项目目录
cd NeuraMind-Frontend

# 如果原项目使用 npm，迁移步骤：
rm -rf node_modules package-lock.json
pnpm install

# 锁定包管理器（强制团队统一）
# 在 package.json 中添加：
{
  "packageManager": "pnpm@9.15.0"
}
```

## 三、核心机制理解

### 3.1 内容寻址存储（Content-Addressable Store）

```
~/.pnpm-store/v3/files/
    ├── 00/1a2b3c...   # 根据文件内容哈希命名
    ├── 12/4d5e6f...
    └── ...

你的项目 node_modules/
    ├── .pnpm/         # 真实依赖的硬链接仓库
    └── vue/           # 符号链接指向 .pnpm/vue@3.x/node_modules/vue
```

**好处**：同一台机器上 100 个项目用 `vue@3.4.21`，硬盘里只存一份实体文件。

### 3.2 非扁平 node_modules

pnpm 的 `node_modules` 结构：

```
node_modules/
├── .pnpm/                    # 所有依赖的真实位置
│   ├── vue@3.4.21/
│   │   └── node_modules/
│   │       ├── vue/          # 真实文件（硬链接）
│   │       └── @vue/         # vue 的依赖，不外泄
│   └── pinia@2.1.7/
│       └── node_modules/
│           ├── pinia/
│           └── vue/          # pinia 自己的 vue 依赖，与顶层隔离
├── vue -> .pnpm/vue@3.4.21/node_modules/vue   # 符号链接
└── pinia -> .pnpm/pinia@2.1.7/node_modules/pinia
```

**严格性**：你的代码只能 `import` 写在 `package.json` `dependencies` 里的包，彻底杜绝幽灵依赖。

## 四、常用命令速查表

| 场景               | 命令                                       |
| ------------------ | ------------------------------------------ |
| 安装全部依赖       | `pnpm install`                             |
| 添加运行时依赖     | `pnpm add three`                           |
| 添加开发依赖       | `pnpm add -D vite @vitejs/plugin-vue`      |
| 添加全局工具       | `pnpm add -g @antfu/ni`                    |
| 移除依赖           | `pnpm remove lodash`                       |
| 更新指定包         | `pnpm update three`                        |
| 更新全部包（谨慎） | `pnpm update`                              |
| 运行脚本           | `pnpm dev` / `pnpm build` / `pnpm preview` |
| 查看依赖树         | `pnpm list`                                |
| 查看过时的包       | `pnpm outdated`                            |
| 清理全局存储垃圾   | `pnpm store prune`                         |
| 查看存储路径       | `pnpm store path`                          |

## 五、Monorepo 工作区实践（重点）

适合你的 NeuraMind 项目拆分：`apps/web`（前端）+ `apps/server`（后端）+ `packages/shared`（共享类型/工具）

### 5.1 目录结构

```
NeuraMind/
├── pnpm-workspace.yaml      # 工作区定义
├── package.json             # 根目录脚本 & 共享 devDeps
├── apps/
│   ├── web/                 # Vue3 + Vite 前端
│   │   └── package.json
│   └── server/              # Gin 后端（或 Node 中间层）
│       └── package.json
└── packages/
    ├── shared/              # 共享类型定义、工具函数
    │   └── package.json
    └── ui/                  # 共享组件库（如需）
        └── package.json
```

### 5.2 配置文件

**`pnpm-workspace.yaml`**（根目录）：

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

**根目录 `package.json`**：

```json
{
  "name": "neuramind",
  "private": true,
  "packageManager": "pnpm@9.15.0",
  "scripts": {
    "dev:web": "pnpm --filter web dev",
    "dev:server": "pnpm --filter server dev",
    "build": "pnpm -r build",
    "clean": "pnpm -r exec rm -rf dist node_modules"
  },
  "devDependencies": {
    "@types/node": "^20.0.0",
    "typescript": "^5.4.0"
  }
}
```

### 5.3 包间依赖引用

假设 `apps/web` 需要引用 `packages/shared`：

```bash
# 在 apps/web 目录下执行
pnpm add @neuramind/shared --workspace

# 或在 apps/web/package.json 中手动添加：
# "dependencies": { "@neuramind/shared": "workspace:*" }
```

`workspace:*` 表示始终使用本地工作区最新版本，发布时会自动替换为实际版本号。

### 5.4 常用过滤命令

```bash
# 只在 web 项目执行 dev
pnpm --filter web dev

# 在 shared 和所有依赖它的项目执行 build
pnpm --filter shared... build

# 在所有包执行 test
pnpm -r test

# 只给 web 项目添加依赖
pnpm --filter web add axios
```

## 六、Vue3 + Vite 项目配置建议

### 6.1 `.npmrc` 项目级配置

在项目根目录创建 `.npmrc`，优化 pnpm 行为：

```ini
# 严格对等依赖（防止版本冲突静默通过）
strict-peer-dependencies=false

# 自动安装对等依赖（Vue 生态很多包需要 vue 作为 peer）
auto-install-peers=true

# 使用符号链接（默认，保持兼容性）
node-linker=isolated

#  shamefully-hoist 仅在极端兼容问题时开启（如某些老插件找不到依赖）
# shamefully-hoist=true
```

### 6.2 依赖分类策略

```json
{
  "dependencies": {
    "vue": "^3.4.21",
    "vue-router": "^4.3.0",
    "pinia": "^2.1.7",
    "three": "^0.164.0",
    "axios": "^1.6.0"
  },
  "devDependencies": {
    "vite": "^5.2.0",
    "@vitejs/plugin-vue": "^5.0.0",
    "typescript": "^5.4.0",
    "vue-tsc": "^2.0.0",
    "sass": "^1.75.0",
    "tailwindcss": "^3.4.0"
  }
}
```

**原则**：运行时需要的放 `dependencies`，构建/开发工具放 `devDependencies`。

### 6.3 锁定文件提交

**必须将 `pnpm-lock.yaml` 提交到 Git**。

- 保证团队协作时依赖版本完全一致
- CI/CD 构建时 `pnpm install --frozen-lockfile` 可复现环境

## 七、CI/CD 与部署

### 7.1 GitHub Actions 示例

```yaml
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Install pnpm
        uses: pnpm/action-setup@v3
        with:
          version: 9
      
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'pnpm'          # 自动缓存 pnpm 存储
      
      - name: Install deps
        run: pnpm install --frozen-lockfile
      
      - name: Build
        run: pnpm build
```

### 7.2 Docker 构建优化

```dockerfile
# 多阶段构建，利用 pnpm 缓存
FROM node:20-alpine AS builder
RUN npm install -g pnpm
WORKDIR /app
COPY pnpm-lock.yaml package.json ./
RUN pnpm install --frozen-lockfile
COPY . .
RUN pnpm build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
```

## 八、常见问题排查

| 现象                         | 原因                               | 解决                                                 |
| ---------------------------- | ---------------------------------- | ---------------------------------------------------- |
| `ERR_PNPM_PEER_DEP_ISSUES`   | 对等依赖版本冲突                   | 在 `.npmrc` 加 `strict-peer-dependencies=false`      |
| 某些包找不到依赖             | 包内部硬编码了 `node_modules` 路径 | `.npmrc` 临时开启 `shamefully-hoist=true`            |
| 全局存储占用过大             | 历史版本累积                       | `pnpm store prune` 清理未引用文件                    |
| Windows 上符号链接报错       | 权限不足                           | 以管理员身份运行终端，或开启开发者模式               |
| 切换分支后 `pnpm install` 慢 | lockfile 冲突                      | 删除 `node_modules` 和 `pnpm-lock.yaml` 重新 install |

## 九、团队协作规范

1. **统一版本**：根目录 `package.json` 必须包含 `"packageManager": "pnpm@x.x.x"`
2. **禁止混用**：项目中不允许出现 `package-lock.json` 或 `yarn.lock`，发现即删除
3. **提交锁定文件**：`pnpm-lock.yaml` 必须进 Git
4. **安装命令**：CI 和本地统一使用 `pnpm install --frozen-lockfile`
5. **依赖审计**：定期执行 `pnpm audit` 检查安全漏洞，`pnpm update` 保持依赖新鲜
